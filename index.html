Semantisches HTML

top posted by SG 27.07.2023 11:43 Uhr

Ich bin bei der Recherche zu semantischem HTML auf die Seite https://kulturbanause.de/blog/html-elemente-und-semantik/ gestossen. Hier ein paar Auszüge:

Semantik sieht man nicht

Bei der semantischen Auszeichnung mittels HTML stellt sich nicht die Frage nach dem Aussehen sondern ausschließlich nach der Bedeutung der einzelnen Inhalte. Es ist also ganz schlechter Stil, wenn man beispielsweise eine Überschrift der Gattung 3 <h3> verwendet, nur weil man möchte, dass die Überschrift eine bestimmte Farbe oder Größe bekommt. Eine technisch gut entwickelte Website trennt Gestaltung und Semantik vollständig (siehe standardkonformes Webdesign).

Warum Semantik in HTML wichtig ist

Nun könnte man denken, dass der korrekte Einsatz von HTML-Elementen gemäß ihrer Bedeutung überflüssig ist, da die Bedeutungsebene von den Besuchern der Website nicht gesehen wird. Wichtig ist die semantische Auszeichnung unter anderem, da Suchmaschinen den Quellcode lesen und anhand der verwendeten HTML-Elemente die Wichtigkeit von Inhalten ableiten. Das hat starke Auswirkungen auf das Ranking einer Website. Daher empfiehlt es sich zum Beispiel, relevante Suchbegriffe (Keywords) in semantisch gewichtigeren HTML-Elementen wie dem Seitentitel (<title>) oder der Überschrift 1 (<h1>) unterzubringen.



HTML Character Entities

top posted by SG 27.07.2023 12:03 Uhr

Wie kann man HTML-Tags auf einer Webseite anzeigen lassen?

Um HTML-Tags ohne Interpretation des Code-Editors und des Browsers auf einer Webseite anzeigen zu lassen muss man sogenannte HTML Character Entities benutzen. Dies verhindert das Zeichenfolgen, die sich im Umfang der HTML-Tags befinden, nicht interpretiert werden, sondern einfach nur angzeigt werden.

Beispiel am HTML-Tag body

Wenn man das body-Tag aus HTML in einer Webseite ausschreiben will, benötigt man die HTML Character Entities für "kleiner" und "größer":

& l t ; ist <
& g t ; ist >

Wenn man nun die folgende Zeichenfolge im HTML-Quellcode nutzt, erscheint das HTML-Tag body auf der Webseite:

& l t ;body& g t ; verwandelt sich in <body> und wird nicht vom Browser interpretiert.

Weitere Entities findet man unter https://developer.mozilla.org/en-US/docs/Glossary/Entity

CSS Games

top posted by SG 27.07.2023 12:28 Uhr

Wir haben zu den Themen CSS ein paar Seiten empfohlen bekommen auf denen man sich spielerisch mit dem Thema CSS, insbesondere den Themen "CSS Selektoren", "CSS Grid" und "CSS flexBox" beschäftigen kann.

CSS Diner

Hier kann man sich durch 32 Level arbeiten um ein Verständnis für die verschiedenen Arten der CSS-Selektoren zu erhalten. Es wird immer eine Ansprache von bestimmten Elementen verlangt die man anhand von besimmten Selektoren und deren Verknüpfung erstellen kann.

Link: https://flukeout.github.io/

Grid Garden

Im Laufe dieses Spiels, welches aus 28 Level besteht, wird man Schritt für Schritt an die Möglichkeiten des "CSS Grids" herangeführt. Es werden einem Aufgaben gestellt die eine besimmte Anordnung von Elementen im Grid verlangt.

https://cssgridgarden.com/#de

Flexbox Froggy

In diesem Spiel muss man mit CSS-Eigenschaften aus dem Bereich "CSS flexBox" Frösche auf den richtigen Platz positionieren. Je nach Level muss man immer mehr Befehle nutzen um zur Lösung zu gelangen.

https://flexboxfroggy.com/#de

Flexbox Defense

Dieses Spiel, aus dem Genre "Tower Defense", beschäftigt sich ebenfalls mit CSS "flexbox". Hier ist die Aufgabe die Platzierung der Verteidigungsanlagen um die Wellen an Zielen zu zerstören.

http://www.flexboxdefense.com/ (kein SSL!)
